# 📸 图片管理最佳实践指南

## 🎯 图片命名规范

### 标准命名格式
```
日期-文件名-序号.jpg
```

### 示例
```
2026-02-09-社区入口设计观察-1.jpg
2026-02-09-社区入口设计观察-2.jpg
2026-02-09-社区入口设计观察-3.jpg
2026-02-09-社区入口设计观察-4.jpg
```

### 命名要素解析
```
2026-02-09     → 观察日期（与笔记文件名一致）
-              → 分隔符，便于解析
社区入口设计观察 → 观察主题/文件名
-1            → 图片序号（从1开始）
.jpg           → 文件格式
```

## 📁 存储位置结构

### 推荐目录结构
```
community-observation-web/
├── images/
│   └── observations/
│       ├── 2026-02-09-社区入口设计观察-1.jpg
│       ├── 2026-02-09-社区入口设计观察-2.jpg
│       ├── 2026-02-09-社区入口设计观察-3.jpg
│       └── 2026-02-09-社区入口设计观察-4.jpg
├── content/
│   └── observations/
│       └── 2026-02-09 社区入口设计观察.md
└── docs/
    └── dev-logs/
        └── 2026-02-09.md
```

### 为什么这样存储？
1. **网站自动识别**：Next.js 会读取 `/images/observations/`
2. **与模板匹配**：模板已预设这个路径
3. **便于管理**：日期和主题清晰分类
4. **URL 稳定**：相对路径不会变化

## 🖼️ Obsidian 中插入图片的方法

### 方法一：Markdown 语法（推荐）
```markdown
![图片描述](/images/observations/2026-02-09-社区入口设计观察-1.jpg)
```

### 方法二：Obsidian 拖拽
1. 打开文件夹面板
2. 将图片文件拖拽到笔记中
3. 自动生成 Markdown 链接

### 方法三：Obsidian 粘贴
1. 复制图片文件
2. 在笔记中 `Ctrl+V` 粘贴
3. 选择"复制到指定文件夹"
4. 选择 `images/observations/`

### 方法四：使用模板预设
模板已自动生成图片路径，直接使用：
```markdown
![观察对象全景](/images/observations/2026-02-09-社区入口设计观察-1.jpg)
![关键细节特写](/images/observations/2026-02-09-社区入口设计观察-2.jpg)
![使用场景展示](/images/observations/2026-02-09-社区入口设计观察-3.jpg)
![问题特写](/images/observations/2026-02-09-社区入口设计观察-4.jpg)
```

## 🎮 Obsidian 图片插件推荐

### 1. Image auto upload Plugin
- **功能**：自动上传图片到图床
- **适用**：大量图片管理
- **设置**：配置图床 API

### 2. Paste image into note Plugin
- **功能**：粘贴图片自动保存
- **适用**：快速截图插入
- **设置**：指定保存路径

### 3. Excalidraw Plugin
- **功能**：图片标注和绘图
- **适用**：图片分析和说明
- **设置**：集成 Markdown 渲染

## 🔍 图片质量优化

### 推荐设置
- **格式**：JPG（照片） / PNG（图表）
- **分辨率**：1920x1080（全高清）
- **压缩**：保持质量的前提下减小文件大小
- **命名**：英文+数字，避免特殊字符
